# Math Digital Workflow

## Overview

This guide is built for math majors who are moving into more theoretical and proof based courses. At this level, handwritten notes and static graphs start to become inefficient. It becomes harder to organize work, revisit concepts, and visualize ideas clearly.

The purpose of this guide is to introduce a structured digital workflow that improves how math is learned, written, and applied. This includes tools for note taking, computation, graphing, and version control.

---

## Who This Guide Is For

This guide is for math majors who:
- Are taking or planning to take proof based or upper level math courses
- Want to improve how they organize and store their work
- Are interested in learning how programming can support mathematics
- Have basic familiarity with tools like GitHub, LaTeX, or a terminal

This is not a beginner guide to programming. Instead, it builds on basic knowledge and shows how to connect everything into one system.

---

## Problem This Guide Solves

Many math students run into the same issues:
- Notes are scattered between notebooks, files, and random documents
- Graphs are static and cannot be easily modified or reused
- Computations are done by hand when they could be automated
- There is no system for organizing or revisiting past work

This guide provides a solution by creating a repeatable workflow that allows you to:
- Digitize and structure notes
- Program and visualize graphs
- Use code to support problem solving
- Keep all work organized and version controlled

---

## Tools Covered

This guide will go into detail on the following tools and how they work together:

- LaTeX and Overleaf for writing mathematical notes and proofs
- Obsidian for organizing notes and building a connected knowledge system
- R for computation and as a calculator for applied math and statistics
- Python for numerical methods and scripting
- NumPy for working with arrays and mathematical operations
- Matplotlib for graphing and visualization
- Git and GitHub for version control and storing work
- Terminal navigation for managing files and running programs

Each section will focus on practical use, not just theory.

---

## What You Will Learn

By following this guide, you will learn how to:
- Write clean and structured math notes using LaTeX
- Organize notes in a way that makes them easy to revisit
- Program graphs instead of drawing them manually
- Use R and Python to solve problems more efficiently
- Track changes in your work using Git
- Build a workflow that saves time and reduces confusion

---

## Assumptions

This guide assumes that you:
- Understand basic math notation
- Have seen LaTeX before, even at a basic level
- Know what GitHub is and have used it at least once
- Are comfortable navigating a computer file system

If not, you can still follow along, but you may need to spend extra time getting familiar with these tools.

---

## How This Guide Is Organized

The guide is structured as a workflow, not isolated tools.

1. Foundations  
   Setting up your environment and understanding the role of each tool  

2. Writing and Notes  
   Using LaTeX, Overleaf, and Obsidian to create structured notes  

3. Computation  
   Using R and Python to support mathematical work  

4. Visualization  
   Creating graphs and visual outputs using code  

5. Version Control  
   Using Git and GitHub to manage and store your work  

6. Workflow Integration  
   Connecting everything into one repeatable system  

---

## Tips for Using This Guide

- Do not try to learn everything at once  
- Focus on building a simple workflow first  
- Practice by applying tools to your current classes  
- Revisit sections as your courses become more advanced  
- Keep your system consistent  

At first, this may seem like a lot. Once you get used to it, it becomes a natural part of how you approach math.

---

## Goal

The goal of this guide is simple.

To help math majors move from disorganized and manual work to a system that is structured, efficient, and scalable.

Once built, this workflow allows you to focus less on managing your work and more on understanding mathematics.
